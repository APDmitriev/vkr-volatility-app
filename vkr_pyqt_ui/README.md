# VKR PyQt UI

PyQt-клиент для backend ВКР по анализу и прогнозированию волатильных временных рядов.

## Запуск интерфейса

```powershell
venv\Scripts\python.exe main.py
```

## Зависимости

```powershell
venv\Scripts\python.exe -m pip install -r requirements.txt
```

## Как подключить backend

1. Запусти backend отдельно, например:

```powershell
uvicorn app.main:app --reload
```

2. Убедись, что backend отвечает по адресу `http://127.0.0.1:8000`.
3. В интерфейсе открой **Настройки**.
4. Выбери режим `rest`.
5. Укажи `Base URL`: `http://127.0.0.1:8000`.
6. Нажми **Проверить backend** или **Сохранить настройки**.

Префикс `/api/v1` добавляется автоматически.

## Что реально подключено к backend

- список и создание проектов;
- загрузка датасета через `POST /api/v1/datasets/upload`;
- валидация датасета;
- предобработка датасета;
- расчёт волатильности;
- ARIMA-прогноз;
- подбор ARIMA;
- GARCH-прогноз;
- Fuzzy First Order forecast.

## Важные ограничения текущего backend

Backend пока **не содержит**:

- отдельного endpoint для общего EDA-анализа;
- endpoint для списка экспериментов;
- endpoint для Linear Regression / Random Forest.

Поэтому в UI:

- общая статистика, тренд и ACF считаются на стороне клиента по обработанному датасету;
- таблица экспериментов и страница сравнения работают по результатам текущей сессии клиента;
- в режиме `rest` доступны модели: **ARIMA, ARIMA Auto, GARCH, Fuzzy First Order**.


## Важно про проекты
По умолчанию UI может стартовать в режиме `mock`. В этом режиме проекты создаются только локально в интерфейсе. Чтобы сохранять проекты в реальном backend, откройте `Настройки`, выберите `backend_mode = rest`, укажите `Base URL`, нажмите `Сохранить настройки`, затем перезапустите приложение или заново откройте страницу `Проекты`. Настройки сохраняются в файле `ui_settings.json`.

## История экспериментов

В интерфейс добавлен отдельный раздел **История экспериментов**.

После каждого успешного запуска обучения модель сохраняется в локальный журнал `ui_experiments.json` рядом с `main.py`. В запись попадают:

- идентификатор эксперимента;
- дата запуска;
- проект;
- датасет;
- модель;
- параметры модели;
- метрики MAE, MSE, RMSE, MAPE;
- путь к файлу результата, если backend его вернул;
- технические ключи ответа backend.

Историю можно:

- просматривать в таблице;
- смотреть детали выбранного эксперимента;
- удалять отдельные записи;
- очищать историю текущего проекта;
- экспортировать в CSV.

Важно: текущий backend ВКР выполняет расчёты, но не содержит отдельного API для журнала экспериментов. Поэтому постоянное хранение истории реализовано на стороне PyQt-клиента.
